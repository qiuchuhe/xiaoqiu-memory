# 2026-07-01 全天工作备份

## 主要事件

### 凌晨自动重启诊断
- 用户发现电脑自动重启（不是睡眠），洛克王国等自启动程序弹出
- 系统日志显示：凌晨 2:34 系统崩溃，3:05 TrustedInstaller 触发重启
- LastBootUpTime: 2026/7/1 3:05:44

### 根因分析
三层原因：
1. **直接触发**：powercfg.exe 在系统空闲时尝试电源状态转换
2. **中层原因**：Windows Update 后台服务（Store应用更新 + Defender病毒库更新）不受「暂停更新」策略约束，仍在自动安装
3. **冲突结果**：Windows Update 活跃期 + 电源转换 → 资源死锁 → 系统挂死 → 硬件看门狗强制重启

证据链：
- Event 41 (Kernel-Power): 非正常关机
- Event 6008: 上次关机 2:34:07 是意外的
- Event 1074: TrustedInstaller.exe 触发重启，原因「操作系统:重新配置(计划内)」
- Event 19 ×3: Store 应用自动更新成功 (Xbox, People)
- Event 44: Windows Update 下载 Defender KB2267602

### 修复措施（已执行）
1. **服务禁用**：
   - `wuauserv` (Windows Update) → Disabled
   - `UsoSvc` (Update Orchestrator) → Disabled
   - `WaaSMedicSvc` → Manual（不会自动启动）
   - `BITS` → Manual（不会自动启动）

2. **注册表设置**：
   - `AUOptions = 2`（通知后才下载，不自动安装）
   - `NoAutoRebootWithLoggedOnUsers = 1`（已登录时禁止自动重启）
   - `Store AutoDownload = 2`（禁止 Store 自动更新）

3. **电源配置**（Performance 计划）：
   - 睡眠超时：AC/DC 均为 从不
   - 休眠超时：AC 从不，DC 1小时

### 用户要求
- 电脑可以正常睡眠
- 但不能因后台服务冲突导致自动重启
- 保持 24/7 运行
