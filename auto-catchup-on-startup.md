# 自动补扫规则

## 触发条件

用户打开 Claude Code 时，**同时满足以下条件**则自动执行补扫：

1. 当前时间是 **周一至周五**（A股交易日）
2. 当前时间在 **8:00 - 15:00** 之间
3. 当前会话是当天的**第一次启动**（不是恢复已有会话）

## 执行流程

### Step 1: 判断已过窗口
对照当前时间，列出所有已过时间点的窗口：
- 8:00 → 早间外盘
- 9:25 → 竞价扫描
- 9:36 → 低开回升
- 10:03 → 早盘主攻
- 13:07 → 午盘开盘
- 14:33 → 尾盘决战

### Step 2: 并行执行两件事
```bash
# A. 运行本地补扫脚本（scanner + monitor + bounce）
python "D:\AI小秋\策略量化\策略1\catchup_market.py"

# B. 同时WebSearch搜索（放在catchup脚本运行期间并行执行）
# - "美股 {昨日日期} 收盘 纳斯达克 道琼斯 标普 费城半导体"
# - "隔夜要闻 {今日日期} 重大事件"
```

### Step 3: 读取结果 + 分析
- 读取 `D:\AI小秋\策略量化\策略1\catchup_result.json`
- 读取 `D:\AI小秋\策略量化\策略1\catchup_report.md`
- 结合 WebSearch 外盘数据

### Step 4: 合并输出一站式报告

```
═══════════════════════════════════════════
📊 交易时段补扫报告  {日期} {星期}
触发时间: {当前时间} · 已过窗口: {N}个
═══════════════════════════════════════════

【8:00 早间外盘】→ WebSearch补搜
[美股收盘 + 中概 + 隔夜要闻摘要]

【9:25 竞价扫描】→ scanner.py 结果
策略1信号: X只 | 竞价方向判断

【9:36 低开回升】→ tech_bounce_scan 结果
[跌停翘起 + 低开回升TOP5 + 板块联动]

【10:03 早盘主攻】→ scanner + monitor 结果
[与前两轮对比，哪些走强/走弱]

【13:07 午盘开盘】→ scanner 结果（如已过）

───────────────────────────────────────────
🎯 综合判定（过朋友框架）
- 热门题材？龙头？低吸？
- 今日操作建议: 空仓 / 关注 / 准备下手
- 候选股: 0-3只 + 理由
───────────────────────────────────────────
```

## 已过窗口的判断规则

| 错过窗口数 | 策略 |
|------------|------|
| 0个 (8:50打开) | 提示"等待9:25竞价"，不补扫 |
| 1-2个 | 全部补扫 + 合并输出 |
| 3-4个 | 全部补扫 + 重点对比多轮信号变化 |
| 5-6个 | 全部补扫 + 重点放在"现在该怎么做"而非回顾 |

## 注意事项

- **只在交易日触发**，周末/节假日跳过
- **数据时效性说明**: 补扫用的是当前实时数据（非历史），需标注"当前重跑，非当时快照"
- **如果 15:00 之后打开**：不触发补扫，直接提示"今日收盘，等待晚间掘金(21:57)"
- **如果 catchup_market.py 不存在或报错**：回退到手动逐个运行 scanner.py / monitor.py
- 补扫完成后，**自动覆盖**后续定时窗口（不需要再等cron触发）

**Why:** 用户因睡眠问题无法准时在 9:25/9:36/10:03 打开 Claude，导致错过所有扫描。此规则确保无论何时打开，都能自动补上所有已过窗口的数据和分析。
**How to apply:** 每次会话启动时自动检查。[[trading-cron-jobs]] [[morning-routine]] [[trading-advice-rule]]
